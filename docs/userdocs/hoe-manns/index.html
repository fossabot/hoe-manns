Introduction to hoe-manns #

hoe-manns is a extension of the Ruby rake/rubygems helper for project Rakefiles hoe (see https://github.com/seattlerb/hoe (https://github.com/seattlerb/hoe).) It extends the standard Ruby tasks to some new useful tasks.

To use the software you will need a installed Ruby version (Higher or equal 2.2.0), bundler-audit and a installed and already configured hoe.

Installation #

For installing you have to type in your usershell: gem install hoe-manns.

How to use Ruby in your projects #

First you have to open the Rakefile of your project and writes there near the Hoe.spec line:

Hoe.plugin :manns

Hoe.spec 'yourproject' do ...

end

Sure now you have to add hoe-manns as dependency in your Rakefile and generate a new Gemfile now. Then hoe-manns is ready to work.

Usage #

hoe-manns integrates some additional Rake-Taks as explained in the next table.

Table 1: The Rake Tasks #
TaskEffect
rake bundler:gemfile_lock updates a Gemfile.lock (does a 'bundle update')
rake bundler:gemfile_install generates a new Gemfile.lock (does a 'bundle install')
rake bundle-audit:run downloads the actual security warnings from bundle-audit and checks possible problems by the dependencies.
rake clean-pkg checks if a "pkg" directory is present, if then it will be deleted.
rake copy_master it expects a branch "develop" and merges the new stuff into the master branch. Then it runs a rake git:tag and returns again into the develop branch.
rake remove_pre_gemspec checks the presence of a Pre-Gemspec (a generated file for test cases) and deletes them.
rake run_before_release runs the task in that order: git:manifest, bundler:gemfile, bundler:gemfile_lock, gem:spec_remove, bundle_audit:run, copy_master.

⁠4. Usage

hoe-manns integrates some additional Rake-Taks as explained in the next table.
⁠

Table 1. The Rake Tasks

TaskEffect
rake bundler:gemfile_lockgenerates a new Gemfile.lock
rake bundle-audit:rundownloads the actual security warnings from bundle-audit and checks possible problems by the dependencies.
rake clean-pkgchecks if a "pkg" directory is present, if then it will be deleted.
rake copy_masterit expects a branch "develop" and merges the new stuff into the master branch. Then it runs a rake git:tag and returns again into the develop branch.
rake remove_pre_gemspecchecks the presence of a Pre-Gemspec (a generated file for test cases) and deletes them.
rake run_before_releaseruns the task in that order: git:manifest, bundler:gemfile, bundler:gemfile_lock, gem:spec_remove, bundle_audit:run, copy_master.
rake run_after_releaseruns the tasks: send_email and clean_pkg
